# ImageToUMG 开发计划（Roadmap）

本文档记录 ImageToUMG 项目的开发路线与阶段性目标。

---

## Phase 1：基础框架搭建

- [x] 创建 Unreal Engine 5.7 项目
- [ ] 配置 C++ 模块编译环境
- [ ] 建立基础项目目录规范
- [ ] 初始化版本控制与忽略规则

---

## Phase 2：MCP 通信层

- [ ] 调研并集成 MCP SDK
- [ ] 实现 MCP Server 基础服务
- [ ] 定义工具接口（Tools）标准
- [ ] 完成与 AI 客户端的握手测试

---

## Phase 3：图片解析与识别

- [ ] 集成图像处理库 / AI 视觉模型
- [ ] 实现 UI 控件类型识别（按钮、文本、图片、输入框等）
- [ ] 实现控件位置与尺寸检测
- [ ] 提取颜色、字体、间距等样式信息

---

## Phase 4：UMG 蓝图生成

- [ ] 开发 UMG Widget 蓝图创建工具
- [ ] 还原控件层级嵌套结构
- [ ] 自动设置控件基础属性（位置、大小、锚点）
- [ ] 支持常用控件（Text、Image、Button、CanvasPanel 等）

---

## Phase 5：样式映射与微调

- [ ] 建立样式映射规则（颜色 → SlateColor、字体 → Font 等）
- [ ] 支持自定义样式模板
- [ ] 提供生成后的微调接口
- [ ] 处理边界情况（如自适应布局）

---

## Phase 6：完善与发布

- [ ] 编写使用示例与演示场景
- [ ] 完善错误处理与日志系统
- [ ] 性能优化与内存管理
- [ ] 撰写完整使用文档
- [ ] 发布首个可用版本

---

## 长期展望

- [ ] 支持从 Figma / Sketch 等设计文件直接导入
- [ ] 支持响应式/自适应布局自动生成
- [ ] 集成更多 UE5 UI 功能（如动画、绑定）
- [ ] 开源 MCP Server 独立部署方案
